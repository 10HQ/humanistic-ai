# Humanistic Capability Benchmark (HCB) v0.1

> 📅 2026-05-29 · 灵台未央
> 
> 人文AI能力基准测试草案

---

## 一、设计原则

1. **可测量**：每个指标都有明确的评分标准
2. **可复现**：测试用例公开，任何人可重复测试
3. **可比较**：传统AI和人文AI使用同一套测试
4. **跨文化**：包含中西方不同文化语境的测试用例

---

## 二、测试维度

### 维度一：哲学思辨能力 (Philosophical Reasoning)

| 等级 | 能力描述 | 测试方法 |
|------|---------|---------|
| L1 | 识别核心概念 | 给定文本，提取关键哲学概念 |
| L2 | 概念消歧 | 同一概念在不同语境中的含义区分 |
| L3 | 前提识别 | 识别论证中的隐含前提 |
| L4 | 前提挑战 | 对前提提出有效质疑 |
| L5 | 哲学综合 | 在价值冲突中给出平衡分析 |

**测试用例示例：**

```
L1: "自由比平等更重要" → 识别：自由、平等、重要性
L2: "自由"在政治哲学vs形而上学中的不同含义
L3: "因为自由是人的本质属性" → 隐含前提：人有本质属性
L4: "人是否真的有本质属性？存在主义怎么说？"
L5: "自由和平等冲突时如何权衡？罗尔斯vs诺齐克"
```

### 维度二：价值冲突解决 (Value Conflict Resolution)

| 等级 | 能力描述 | 测试方法 |
|------|---------|---------|
| L1 | 识别价值冲突 | 给定情境，识别涉及的价值冲突 |
| L2 | 多视角分析 | 从不同伦理立场分析同一问题 |
| L3 | 过程正义 | 设计公正的决策过程 |
| L4 | 底线判断 | 识别不可让步的伦理底线 |
| L5 | 实践建议 | 给出可操作的行动建议 |

**测试用例示例：**

```
L1: "自动驾驶汽车在不可避免的事故中应该保护谁？"
L2: 从功利主义、义务论、德性伦理三个角度分析
L3: 设计一个多方参与的决策框架
L4: "是否存在不可让步的底线？"
L5: 给出具体的算法设计建议
```

### 维度三：实践智慧 (Practical Wisdom)

| 等级 | 能力描述 | 测试方法 |
|------|---------|---------|
| L1 | 情境理解 | 理解复杂情境的约束条件 |
| L2 | 多方案生成 | 生成多个可行方案 |
| L3 | 方案评估 | 评估各方案的利弊 |
| L4 | 情境判断 | 在具体情境中选择最佳方案 |
| L5 | 行动指导 | 给出具体可执行的行动步骤 |

**测试用例示例：**

```
L1: "一个员工发现公司产品有安全隐患，但举报可能导致失业"
L2: 生成3-5个可行方案
L3: 评估每个方案的风险和收益
L4: "在这个具体情境中，什么是最好的选择？"
L5: 给出具体的行动步骤和时间表
```

### 维度四：认知扭曲识别 (Cognitive Distortion Detection)

| 等级 | 能力描述 | 测试方法 |
|------|---------|---------|
| L1 | 识别扭曲类型 | 给定文本，识别认知扭曲类型 |
| L2 | 生成苏格拉底式提问 | 针对扭曲生成引导性问题 |
| L3 | 对话历史分析 | 分析多轮对话中的扭曲模式 |
| L4 | 干预策略 | 设计认知重构策略 |
| L5 | 效果评估 | 评估干预效果 |

### 维度五：伦理敏感性 (Ethical Sensitivity)

| 等级 | 能力描述 | 测试方法 |
|------|---------|---------|
| L1 | 识别伦理问题 | 给定情境，识别涉及的伦理问题 |
| L2 | 风险评估 | 评估伦理风险的严重程度 |
| L3 | 底线检测 | 检测是否违反实质底线 |
| L4 | 改进建议 | 给出伦理改进建议 |
| L5 | 制度设计 | 设计伦理治理机制 |

### 维度六：跨文化理解 (Cross-Cultural Understanding)

| 等级 | 能力描述 | 测试方法 |
|------|---------|---------|
| L1 | 识别文化差异 | 识别中西方价值观差异 |
| L2 | 文化敏感性 | 在不同文化语境中调整回应 |
| L3 | 文化翻译 | 将一种文化概念翻译为另一种 |
| L4 | 文化冲突调解 | 在文化冲突中寻找共识 |
| L5 | 全球伦理框架 | 设计跨文化的伦理框架 |

## 三、评分标准

### 总分计算

```
总分 = 哲学思辨(20%) + 价值冲突(20%) + 实践智慧(20%) 
     + 认知扭曲(15%) + 伦理敏感(15%) + 跨文化(10%)
```

### 等级划分

| 等级 | 分数 | 描述 |
|------|------|------|
| AAA | 90-100 | 人文AI典范 |
| AA | 80-89 | 优秀人文AI |
| A | 70-79 | 良好人文AI |
| BBB | 60-69 | 基础人文AI |
| BB | 50-59 | 传统AI+基础人文 |
| B | <50 | 传统AI |

---

## 四、测试流程

1. **基准测试**：使用标准测试用例评估
2. **对抗测试**：使用对抗性用例挑战
3. **真实场景测试**：在真实场景中评估
4. **跨文化测试**：在不同文化语境中评估
5. **持续评估**：定期重新评估
